# Frontend Design

Create distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction.

- Purpose: Identify what problem the interface solves and who uses it.
- Tone: Pick an extreme and commit to it, such as brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, or industrial/utilitarian.
- Constraints: Respect framework, performance, accessibility, and technical requirements.
- Differentiation: Decide what makes the design unforgettable and what one element someone will remember.

Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism both work. The key is intentionality, not intensity.

Then implement working code in the requested stack, such as HTML/CSS/JS, React, or Vue, that is:

- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear aesthetic point of view
- Meticulously refined in every detail

## Frontend Aesthetics Guidelines

Focus on the following design levers.

### Typography

- Choose fonts that are beautiful, unique, and interesting.
- Avoid generic fonts like Arial and Inter.
- Prefer distinctive display fonts paired with refined, readable body fonts.
- Make type feel intentional, characterful, and specific to the concept.

### Color and Theme

- Commit to a cohesive aesthetic and define reusable CSS variables.
- Use dominant colors with sharp accents instead of timid, evenly distributed palettes.
- Let color support the concept rather than following default SaaS patterns.

### Motion

- Use animation to create memorable moments and clear interaction feedback.
- Prefer CSS-first motion for HTML when possible.
- Use Motion for React when available.
- Prioritize high-impact moments such as staggered page reveals, scroll-triggered transitions, and surprising hover states.

### Spatial Composition

- Favor unexpected layouts over safe defaults.
- Use asymmetry, overlap, diagonal flow, grid-breaking moments, and deliberate negative space or controlled density.
- Let the layout reinforce the chosen visual direction.

### Backgrounds and Visual Details

- Build atmosphere and depth rather than defaulting to flat solid fills.
- Use effects that match the concept, such as gradient meshes, noise, geometric patterns, layered transparencies, decorative borders, grain overlays, custom cursors, or dramatic shadows.
- Make decorative details feel designed, not pasted on.

## Avoid Generic AI Aesthetics

Never default to:

- Overused font families such as Inter, Roboto, Arial, or generic system stacks
- Cliched purple gradients on white backgrounds
- Predictable hero-section layouts or cookie-cutter dashboard patterns
- Recycled component styling that lacks context-specific character

Interpret creatively and make unexpected choices that feel genuinely designed for the context. Vary between light and dark themes, different fonts, different aesthetics, and different compositional systems. Do not converge on the same design choices across outputs.

## Execution Standard

- Match implementation complexity to the aesthetic vision.
- Maximalist concepts should include richer motion, layering, and visual systems.
- Minimalist or refined concepts should show restraint, precision, spacing discipline, and subtle details.
- Favor elegance through strong execution, not filler.

Remember that extraordinary creative work is possible when the concept is specific and fully committed. Do not hold back. Show what can be created when the design direction is intentional, distinctive, and well executed.
